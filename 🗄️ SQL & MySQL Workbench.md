# 🗄️ SQL 
### 🗄️ SQL & Database Analysis Fundamentals

## Overview

This project demonstrates the use of SQL to query, clean, modify, and analyse relational database data. Using SQL commands and database concepts, I explored structured datasets to extract insights, manage data quality, and perform analytical calculations.

The project showcases core SQL skills used in data analysis and database management, including querying data, filtering records, string matching, date and time functions, aggregate calculations, aliasing, subqueries, logical operators, handling NULL values, and modifying database records.

This project also includes an Entity Relationship Diagram (ERD) to demonstrate understanding of database structure, relationships between tables, primary keys, and foreign keys.

## Objectives

The aim of this project was to:

- Understand and analyse relational database structures
- Write SQL queries to retrieve meaningful information
- Clean and maintain data within database tables
- Identify and handle missing or NULL values
- Modify existing data using SQL commands
- Apply filtering techniques to analyse specific records
- Perform calculations using aggregate functions
- Analyse data using text and date-based queries
- Use advanced SQL techniques such as aliases and subqueries
- Understand relationships between database tables using an ERD

## Database Structure

Before querying the data, I analysed the database structure using an Entity Relationship Diagram (ERD).

The ERD helped identify:

- Database tables and their purpose
- Primary keys used to uniquely identify records
- Foreign keys linking related tables
- Relationships between different data entities
- How tables interact when creating queries

Understanding the database structure allowed accurate querying and improved data analysis.

## Data Preparation & Cleaning

Before performing analysis, I prepared the dataset by:

- Reviewing database tables and column structures
- Checking data consistency and accuracy
- Identifying missing values and NULL entries
- Updating incorrect or incomplete records
- Modifying data where required
- Ensuring the dataset was suitable for analysis
